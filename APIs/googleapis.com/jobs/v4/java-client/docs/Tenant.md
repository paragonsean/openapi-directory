

# Tenant

A Tenant resource represents a tenant in the service. A tenant is a group or entity that shares common access with specific privileges for resources like jobs. Customer may create multiple tenants to provide data isolation for different groups.

## Properties

| Name | Type | Description | Notes |
|------------ | ------------- | ------------- | -------------|
|**externalId** | **String** | Required. Client side tenant identifier, used to uniquely identify the tenant. The maximum number of allowed characters is 255. |  [optional] |
|**name** | **String** | Required during tenant update. The resource name for a tenant. This is generated by the service when a tenant is created. The format is \&quot;projects/{project_id}/tenants/{tenant_id}\&quot;, for example, \&quot;projects/foo/tenants/bar\&quot;. |  [optional] |
